Architecture Approaches

How should the merchant app and admin panel relate to each other?

A

Unified App — Shared Routes (Recommended)

Both merchant and admin live in the same React Router app. Merchant pages under /app/* use Shopify auth + Polaris. Admin pages under /admin/* use secret key auth + Tailwind.

Pros

  • Single deploy on Vercel
  • Shared Prisma DB & Shopify API helpers
  • Admin can reuse merchant's session tokens to call Shopify APIs on their behalf
  • Simplest to build and maintain

Cons

  • Tailwind CSS needs to coexist with Polaris (scoped to admin routes)
  • Admin and merchant share the same process
B

Separate Admin Server

Admin panel is a completely separate app (Express or standalone React Router) that connects to the same database. Deployed as a separate Vercel project.

Pros

  • Clean separation — no CSS conflicts
  • Can scale independently

Cons

  • Two deploys, two repos or monorepo setup
  • Need to share DB connection and session storage
  • More infrastructure to manage
C

Admin as API-Only + Static Frontend

Admin backend is API routes in the same app (/api/admin/*), but the admin frontend is a static HTML/Tailwind page served from /public. No React Router for admin UI.

Pros

  • Zero CSS conflict — static HTML is fully isolated
  • Single deploy
  • Lightweight admin UI

Cons

  • No client-side routing for admin — page reloads on navigation
  • Harder to build rich interactive features

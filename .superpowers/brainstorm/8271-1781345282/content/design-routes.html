Design: Route Structure & Pages

Section 2 of 3 — What pages exist and what they do

Route Map
MERCHANT ROUTES (Embedded in Shopify Admin)
════════════════════════════════════════════
/app                    → Layout shell (existing, Polaris)
/app/_index             → Theme list page (replaces template demo)
                          • Shows all themes for current shop
                          • Role badge (Main, Unpublished, Demo)
                          • [Download] button → triggers .zip download
                          • [Delete] button → confirm dialog → delete
                          • Cannot delete the live/main theme

/app/themes/:id/download → Action route (no UI)
                          • Fetches all theme files via GraphQL
                          • Bundles into .zip with JSZip
                          • Returns as file download response


ADMIN ROUTES (Standalone, Tailwind CSS)
════════════════════════════════════════════
/admin                  → Layout shell with secret key validation
                          Checks ?key= against ADMIN_SECRET_KEY env
                          All child routes inherit auth

/admin/_index           → Shop list dashboard
                          • Lists all shops that installed the app
                          • Shows shop name, domain, install date
                          • Click shop → go to theme management

/admin/shop/:shopId     → Theme management for a specific shop
                          • Lists all themes for that shop
                          • Role badge (Main, Unpublished, Demo)
                          • [Download] button → triggers .zip download
                          • [Delete] button → confirm → delete
                          • Cannot delete the live/main theme

/admin/shop/:shopId/    → Action route (no UI)
  themes/:id/download     Same logic as merchant download


SHARED
════════════════════════════════════════════
app/services/theme.server.js
  • listThemes(admin)         → GraphQL themes query
  • downloadTheme(admin, id)  → Fetch files + zip bundle
  • deleteTheme(admin, id)    → themeDelete mutation
  • getAdminForShop(shopId)   → Load session, create admin client
